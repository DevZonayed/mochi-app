# CLAUDE.md — Maestro repo guide for the coding agent

This file is read on every coding-agent session in this repo. Keep it short.
For deeper context, the agent should consult `.continuum/STATE.md` (project
memory) and `.claude/skills/` (installed skills).

## Commit policy

- Use **Conventional Commits** — see `.claude/skills/maestro-commit/SKILL.md`
  for the full ruleset (types, scopes, 50/72 line budget, examples drawn from
  this repo's actual `git log`).
- **Never** add `Co-Authored-By: Claude`, `🤖 Generated with Claude Code`,
  `Generated by Claude Code`, or any other AI-attribution trailer. A
  `prepare-commit-msg` hook in `.githooks/prepare-commit-msg` strips them on
  every commit as a safety net, but the agent must not write them in the first
  place — the user's clients read this commit log.
- Commit identity is auto-aligned to the **GitHub-logged-in account** via
  `ensureCommitIdentity` (`apps/desktop/electron/git-identity.ts`) at repo
  open / clone / adopt. It uses `gh api user` and falls back to
  `<login>@users.noreply.github.com` if the operator's email is private. The
  trailer-stripping hook is wired the same way via `ensureGitHooks`.

## Repo layout (one-liner)

pnpm + Turborepo monorepo. `apps/desktop` (Electron+React, the brain), `apps/mobile` (Expo, thin remote), `apps/server` (account-scoped relay). Shared: `packages/design-tokens`, `packages/realtime`.

## When in doubt

Read `.continuum/STATE.md`. It carries the live operator notes (build status,
known gotchas, recent decisions) for the current Maestro work.
